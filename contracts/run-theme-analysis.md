# run-theme-analysis.json

Contract for the weekly run analyzer output generated by `scripts/analyze-feedback-run.mjs`.

Default file path: `data/runs/<runId>/analysis.json`

## Schema

```json
{
  "runId": "2026-04-03T00-00-00Z",
  "generatedAt": "2026-04-03T18:45:10.120Z",
  "totalItems": 312,
  "analyzedItems": 312,
  "analyzedBuckets": 14,
  "intro": "This week, teams repeatedly reported friction around MCP permissions and model controls.",
  "themes": [
    {
      "rank": 1,
      "label": "MCP permissions are hard to debug",
      "summary": "Customers repeatedly hit unclear MCP permission failures and need better diagnostics.",
      "repeatCount": 29,
      "severityBreakdown": { "High": 8, "Medium": 17, "Low": 4 },
      "topFeedbackTypes": ["Friction", "Bug Report"],
      "representativeQuote": "I cannot tell whether this failed because of auth or permissions.",
      "customerAccounts": ["Acme", "BetaCo"],
      "supportingKeys": ["4437:a1b2c3d4"],
      "score": 343.4
    }
  ]
}
```

## Notes

- `themes[]` is already ranked by score and clipped to `ANALYZE_TOP_N` (default 5).
- `score` is deterministic and combines:
  - repeat count (primary signal)
  - severity mix
  - confidence mix
  - feedback type weighting
- Analyzer output is also persisted to Supabase table `feedback_run_summaries` for replay/auditing.
